
### ENT-005: Table `order_items`

```sql
CREATE TABLE order_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  order_id UUID NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
  seat_id UUID NOT NULL REFERENCES seats(id),
  price_snapshot_vnd NUMERIC(14,2) NOT NULL CHECK (price_snapshot_vnd >= 0),
  active BOOLEAN NOT NULL DEFAULT TRUE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(order_id, seat_id)
);
CREATE UNIQUE INDEX uq_active_seat_order_item
  ON order_items(seat_id)
  WHERE active = TRUE;
```

**Purpose**: Seat line items and price snapshot.
**Lifecycle**: immutable price after checkout; `active` becomes false when an order is rejected, cancelled, expired, or a ticket is exchanged away from the seat.

> **Assumption**: Application services update `order_items.active = FALSE` in the same transaction that releases or exchanges the seat.
> **Change trigger**: If true historical overlap is required for analytics, add a separate immutable seat-state history table instead of weakening the active uniqueness rule.
