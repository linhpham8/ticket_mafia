### PR-005: Idempotency And State Guard Contract

- **Purpose**: Prevent duplicate checkout, payment completion, admin confirmation, scan, and exchange mutations.
- **Owns**: backend mutation endpoints and persistence.
- **Public Entrypoints**: API-005, API-006, API-007, API-013, API-014, API-015.
- **Allowed dependencies**: clients send `Idempotency-Key`; backend stores key scoped by user/session/operation.
- **Companion Refs**: ADR-001, SEQ-003, SEQ-004, SEQ-005.

All protected mutation endpoints that alter ticket/order/seat state require `Idempotency-Key`. Backend still enforces transaction state guards and unique constraints; the header is not the only safety control.
