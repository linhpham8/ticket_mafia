### PERSONA-001: Football fan buyer

- **Role**: Customer who buys football tickets for a match.
- **Tech Level**: Medium. Uses mobile apps and web checkout, can follow OTP login and QR transfer instructions.
- **Mindset / Quote**: "Tôi muốn chọn đúng ghế và nhận vé QR rõ ràng trước khi tới sân."
- **Goals**:
  1. See matches currently selling tickets.
  2. Select available seats by concrete seat code.
  3. Receive QR/e-ticket after payment confirmation.
- **Pain Points**:
  1. Offline stadium purchase takes effort and time.
  2. Seat availability is not visible before purchase.
  3. Manual payment confirmation can be unclear without ticket status.
- **Usage Context**:
  - **Tần suất**: Occasional, before football matches.
  - **Môi trường**: Mobile-first, can also use web.
  - **Workflow context**: Browses match, selects seats, transfers money, waits for admin confirmation.
  - **Device ưu tiên**: iOS, Android, User Web.
  - **Offline / connectivity**: Online required.
- **Decision-Making Pattern**: Continues if seat availability and payment instructions are clear.
- **Frustration Threshold**: Abandons flow if selected seats disappear without clear reason or ticket status is unclear.
- **Success Signals**: User can complete purchase and open QR/e-ticket from history.
- **Accessibility / Device Notes**: Seat codes and prices must be readable on mobile.
- **Related Epics / Stories**: EP-001, EP-002, EP-003, EP-006, EP-007; US-001, US-002, US-003, US-004, US-005, US-009, US-010, US-012.

### PERSONA-002: Ticket sales admin

- **Role**: Admin who configures matches, seats, prices, transfer QR, and confirms purchases.
- **Tech Level**: Medium. Comfortable with admin web forms and operational status lists.
- **Mindset / Quote**: "Tôi cần tạo trận, đặt giá, và trả vé đúng cho khách đã chuyển khoản."
- **Goals**:
  1. Create and update matches on sale.
  2. Configure seats and active prices by section/floor/VIP.
  3. Confirm or reject pending customer purchases.
- **Pain Points**:
  1. Manual offline sales are hard to demonstrate as an online product.
  2. Price changes must not corrupt orders already in checkout.
  3. Pending transfers need clear timeout handling.
- **Usage Context**:
  - **Tần suất**: During match setup and purchase confirmation windows.
  - **Môi trường**: Desktop browser on Admin Web.
  - **Workflow context**: Before match sale and while users submit payment completion.
  - **Device ưu tiên**: Admin Web.
  - **Offline / connectivity**: Online required.
- **Decision-Making Pattern**: Uses lists and statuses to decide which transaction to confirm or reject.
- **Frustration Threshold**: Operational risk increases if pending transactions, seat states, or prices are ambiguous.
- **Success Signals**: Admin can configure sale data and issue tickets after confirmation.
- **Accessibility / Device Notes**: Tables and status filters should be clear on desktop.
- **Related Epics / Stories**: EP-004, EP-005; US-006, US-007, US-008.

### PERSONA-003: Gate scan operator

- **Role**: Person or system consumer that submits ticket scan status at stadium gate.
- **Tech Level**: Medium. Needs a simple scan/update result, not a full v1 UI.
- **Mindset / Quote**: "Mỗi vé hợp lệ chỉ được vào cổng một lần."
- **Goals**:
  1. Mark valid QR/e-ticket as scanned once.
  2. Reject repeated or invalid scan attempts.
  3. Keep ticket status accurate for admin/user history.
- **Pain Points**:
  1. Duplicate scan creates entry-control risk.
  2. Cancelled or exchanged tickets must not be accepted.
  3. Demo scope does not include a full scanner app.
- **Usage Context**:
  - **Tần suất**: During match entry window.
  - **Môi trường**: Stadium gate integration context.
  - **Workflow context**: API call updates ticket status.
  - **Device ưu tiên**: Scanner app is out of scope; API behavior only.
  - **Offline / connectivity**: Online required for v1.
- **Decision-Making Pattern**: Accepts only clear success/failure responses.
- **Frustration Threshold**: Cannot tolerate accepting an already scanned ticket.
- **Success Signals**: Valid ticket transitions to scanned once; repeated scan is blocked.
- **Accessibility / Device Notes**: N/A for v1 UI.
- **Related Epics / Stories**: EP-006; US-011.
